XClipper は X (Twitter) 向けの無料オープンソースのウェブクリッパーです。スレッド、投稿、記事を PDF またはクリーンな Markdown としてダウンロードし、Obsidian、リサーチ、AI ワークフロー、オフラインアーカイブに活用できる Chrome 拡張機能です。すべてブラウザ内で動作し、API キー不要、アカウント不要、トラッキングなしです。

ワンクリックでコンテンツをエクスポート:

- Markdown として保存
- PDF としてエクスポート（選択可能なテキスト、クリック可能なリンク、ラスタライズなし）
- Markdown をクリップボードにコピー
- Obsidian に直接送信
- .md ファイルと一緒に画像をローカルへダウンロード

すべてローカル環境で実行されます。バックエンドサービスなし、テレメトリーなし、x.com 以外への外部リクエストなし。

主な機能:

- ツイート、スレッド、引用ツイート、ネストしたスレッド、X Articles（旧 Notes）をエクスポート
- 選択可能なテキスト、クリック可能なリンク、絵文字や非 ASCII 文字の正しい表示に対応した PDF エクスポート
- Obsidian、Logseq、Notion、Hugo など Markdown ベースのワークフローで使えるクリーンな Markdown
- obsidian:// URI スキーム経由のワンクリック「Add to Obsidian」対応
- 著者、handle、日付、ソース URL、コンテンツ種別、エンゲージメント統計を含む豊富な YAML frontmatter
- Obsidian 向けの任意 frontmatter: wikilinked [[@handle]] 著者、Dataview 対応メタデータ、生成されたタイトルと説明
- 埋め込み画像をローカルにダウンロードしてリンク切れを防止
- タイトル、ソースドメイン、プレビュー画像付きのリンクカードを取得
- 選択肢、結果割合、投票総数/ステータス行を含む投票を取得
- 引用ツイートの構造と帰属情報を保持
- 単一ツイートまたはスレッド全体をエクスポート
- x.com 内のインラインエクスポートボタン、ツールバーポップアップ、右クリックコンテキストメニュー
- {date}、{handle}、{slug}、{type} などのプレースホルダーを使ったカスタムファイル名テンプレート
- Obsidian 用の任意の Vault 指定と Vault サブフォルダ対応
- エクスポートした Markdown とメディア用の任意のダウンロードサブフォルダ
- 多言語 UI: 日本語、アラビア語、イタリア語、英語、スペイン語、中国語（簡体字）、ドイツ語、ヒンディー語、フランス語、ペルシャ語、ポルトガル語（ブラジル）、ロシア語
- ライトモードとダークモードに対応

こんな用途に最適:

- Obsidian や PKM のワークフロー
- リサーチと参考資料のアーカイブ
- AI プロンプトと RAG パイプライン
- 検索可能なセカンドブレインの構築
- X の長文コンテンツをオフライン保存

現在の制限:

- 動画と GIF は再生可能なメディアファイルとしてエクスポートされません
- x.com がページ構造を大きく変更すると、一部機能が動作しなくなる可能性があります
- x.com のタブがすでに開いている状態で拡張機能をインストールまたは更新した場合は、エクスポート前にタブを再読み込みしてください。これは未初期化ページでのサイレントな失敗を防ぐための仕様です
