XClipper は、X/Twitter のスレッド、投稿、長文記事を、Obsidian、リサーチ、AI ワークフロー、オフラインアーカイブ向けにクリーンな Markdown へ変換するオープンソースの Chrome 拡張機能です。

ワンクリックでエクスポート:

- Markdown として保存
- Markdown をクリップボードへコピー
- Obsidian へ直接送信
- .md ファイルと同じ場所に画像をローカル保存

すべてブラウザ内でローカルに動作。API キー、アカウント、トラッキング、解析は一切なし。

主な機能:

- ツイート、スレッド、引用ツイート、ネストしたスレッド、長文 X Articles（旧 Notes）をエクスポート
- Obsidian、Logseq、Notion、Hugo など Markdown ベースのワークフローで使えるクリーンな Markdown
- obsidian:// URI スキーム経由のワンクリック「Obsidian に追加」
- 著者、ハンドル、日付、ソース URL、コンテンツタイプ、エンゲージメント統計を含む YAML frontmatter
- 任意の Obsidian 向け frontmatter: [[@handle]] ウィキリンク著者、Dataview 対応メタデータ、生成されたタイトルと説明
- 埋め込み画像をローカル保存してリンク切れを防止
- リンクカードをタイトル、ドメイン、プレビュー画像とともにキャプチャ
- 投票（ポール）を選択肢、結果のパーセンテージ、投票数/ステータス行とともにキャプチャ
- 引用ツイートの構造と帰属を保持
- 単一のツイートまたはスレッド全体をエクスポート
- x.com 内のインラインエクスポートボタン、ツールバーポップアップ、右クリックコンテキストメニュー
- {date}、{handle}、{slug}、{type} などのプレースホルダーによるカスタムファイル名テンプレート
- 任意の Obsidian Vault 指定と Vault サブフォルダ
- Markdown とメディア用の任意のダウンロードサブフォルダ
- 多言語 UI: 英語、スペイン語、ドイツ語、フランス語、イタリア語、ロシア語、日本語、ポルトガル語（ブラジル）、中国語（簡体字）、ヒンディー語、アラビア語、ペルシャ語
- ライトモードとダークモードに対応

向いている用途:

- Obsidian や PKM のワークフロー
- リサーチと参照のアーカイブ
- AI プロンプトと RAG パイプライン
- 検索可能なセカンドブレインの構築
- X の長文コンテンツをオフラインで保存

現在の制限:

- 動画と GIF は再生可能なメディアファイルとしてエクスポートされません
- x.com のページ構造が大きく変更されると、一部の機能が動作しなくなる可能性があります
- x.com のタブが既に開いている状態で拡張機能をインストール／更新した場合、エクスポートする前にタブをリロードしてください — 初期化されていないページでの暗黙のエラーを避けるための仕様です
